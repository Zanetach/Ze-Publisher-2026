## 新增需求

### 需求:长文预览区必须可完整纵向滚动
系统必须在预览展示区内支持长内容的完整纵向浏览，不得出现内容被截断且无法继续向下访问的情况。

#### 场景:基础长文滚动可达底部
- **当** 用户打开包含大量段落与图片的长文预览
- **那么** 预览展示区必须出现纵向滚动能力
- **并且** 用户通过滚轮、触控板或滚动条拖拽必须能够滚动到文档末尾

### 需求:滚动能力必须独立于主题切换
系统在用户切换任意主题后必须保持同等的纵向滚动能力，不得因主题样式变更导致滚动失效或内容高度异常。

#### 场景:主题切换后滚动持续有效
- **当** 用户在同一篇长文中切换不同主题
- **那么** 预览展示区仍必须保持可滚动
- **并且** 用户必须仍能滚动至文档底部

### 需求:布局变化不得破坏滚动容器
系统在工具栏显隐、左右布局切换、窗口宽度变化后，必须保持预览展示区作为独立滚动容器。

#### 场景:工具栏状态切换后的滚动一致性
- **当** 用户在长文预览状态下切换工具栏展开/收起
- **那么** 预览展示区必须继续可滚动
- **并且** 不得出现仅顶部部分可见或滚动距离突变的问题

### 需求:复杂内容类型不得阻塞滚动
系统必须确保图片、代码块、Mermaid 图等复杂内容与普通文本混排时，预览展示区仍可持续纵向滚动。

#### 场景:Mermaid与代码块混排的长文滚动
- **当** 文章包含 Mermaid 图表、长代码块与多张图片
- **那么** 预览展示区必须保持稳定滚动
- **并且** 任何单个内容块都不得导致整体滚动被锁死
