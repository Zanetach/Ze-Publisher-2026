## 新增需求

### 需求:单页主动作约束
系统必须在每个工作区页面提供且仅提供一个主动作（Primary Action），并禁止多个同级主动作并列造成决策冲突。

#### 场景:工作区主动作可识别
- **当** 用户进入任意工作区页面
- **那么** 用户可以明确识别唯一主动作并理解下一步

### 需求:任务链路反馈
系统必须在关键步骤提供状态反馈与下一步建议，并禁止在关键动作后无反馈。

#### 场景:执行关键步骤
- **当** 用户完成一次关键动作（如校验或交付）
- **那么** 系统显示当前状态、阻断项数量与下一步建议

### 需求:错误恢复可执行性
系统必须为关键错误提供可执行恢复动作，并禁止仅显示静态错误描述。

#### 场景:出现关键错误
- **当** 用户在核心链路中遇到错误
- **那么** 系统提供可立即执行的恢复操作（如重试、跳转修复、降级方案）
