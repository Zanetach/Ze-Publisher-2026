## 新增需求

### 需求:任务导向工作区布局
系统必须提供 `Prepare`、`Review`、`Deliver` 三个顶层工作区，并禁止在默认模式下以平铺功能清单替代任务域导航。

#### 场景:默认进入任务工作区
- **当** 用户打开 ZePublish 主界面
- **那么** 系统显示三工作区导航并进入默认工作区

### 需求:渐进式布局灵活能力
系统必须在保持默认稳定布局的前提下提供可折叠与可记忆能力，并禁止无约束自由布局导致主动作不可达。

#### 场景:记忆用户布局偏好
- **当** 用户调整可折叠区域并重新打开视图
- **那么** 系统恢复上次保存的布局偏好且主动作可见
